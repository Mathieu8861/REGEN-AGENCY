3Structure des fichiers

projet/
├── index.html ← Page principale
├── [autres-pages].html ← Pages secondaires
├── style.css ← CSS unique (desktop + mobile)
├── script.js ← JavaScript (optionnel)
├── assets/
│ ├── images/
│ └── icons/ ← SVG
├── favicon.png
└── README.md
Règle d'or : 1 seul fichier CSS
Moins de requêtes HTTP, pas de conflits de spécificité, media queries à la fin du fichier.

5Conventions CSS - BEM

Type Syntaxe Exemple
Bloc .bloc .card, .header, .hero
Élément .bloc__element .card__title, .header__logo
Modificateur .bloc--modificateur .btn--primary, .card--featured
État .is-* ou .has-* .is-visible, .has-error

Ordre des propriétés

.element {
    /* 1. Positionnement */
    position: relative;
    z-index: 10;

    /* 2. Display & Box Model */
    display: flex;
    width: 100%;
    padding: var(--spacing-md);

    /* 3. Visuel */
    background: var(--color-white);
    border-radius: var(--radius-md);

    /* 4. Typographie */
    font-family: var(--font-body);
    color: var(--color-dark);

    /* 5. Animations */
    transition: 0.3s ease;
}

6Responsive Design

Breakpoint Media Query Usage
Desktop small @media (max-width: 1200px) Écrans moyens
Tablette @media (max-width: 1024px) iPad, tablettes
Mobile large @media (max-width: 768px) Smartphones grands
Mobile small @media (max-width: 480px) Smartphones petits
Règles :
  • Desktop first : styles par défaut = desktop
  • Media queries en fin de fichier, regroupées par breakpoint
  • Safari : doubler backdrop-filter avec -webkit-backdrop-filter
  • iPhone X+ : utiliser env(safe-area-inset-*)

8Règles strictes

TOUJOURS faire

  • overflow: visible sur sections
  • Doubler backdrop-filter avec webkit
  • Tester sur mobile réel
  • Utiliser rem pour les fonts
  • Safe areas iPhone
  • Alt sur les images

JAMAIS faire

  • overflow: hidden sur sections
  • Header collapse sur mobile
  • Font icons (FontAwesome)
  • Frameworks CSS
  • jQuery
  • Pixels pour les fonts

9Brandboard - Livrable client

Avant de coder, générer un brandboard PDF à envoyer au client avec la maquette.

CONTENU DU BRANDBOARD
En-tête Logo + nom projet + description du style
Palette couleurs 5 couleurs avec codes HEX et usage
Typographies Police titres + police corps (Google Fonts)
Hiérarchie typo H1, H2, H3, body, small avec tailles
Boutons Primary, secondary, outline, ghost
Architecture Liste des pages prévues
Fichier : BRANDBOARD-[NOM-PROJET].html
Pour le PDF : Ouvrir dans navigateur → Imprimer → Enregistrer en PDF

11CSS obligatoire pour l'impression

Pour les fichiers HTML destinés à être exportés en PDF (brandboards, docs), toujours inclure ces règles :

* {
    -webkit-print-color-adjust: exact !important;
    print-color-adjust: exact !important;
    color-adjust: exact !important;
}

@media print {
    body {
        -webkit-print-color-adjust: exact !important;
        print-color-adjust: exact !important;
    }
}
Pourquoi ? Par défaut, les navigateurs n'impriment pas les couleurs de fond (background-color) pour économiser l'encre. Ces règles forcent l'impression des couleurs.

12Initialisation nouveau projet

Dès qu'un nouveau projet est lancé, Claude doit faire ces étapes EN PREMIER :

  1. Créer le dossier site/ avec la structure standard (HTML, CSS, JS, assets/)
  2. Initialiser Git (git init)
  3. Créer le repo GitHub (commit initial + push)
  4. Demander à l'utilisateur d'aller sur Vercel pour importer le repo (seule étape manuelle)
Pourquoi ? Chaque modification sera visible en ligne immédiatement. Le client peut suivre l'avancement en temps réel.

13Livraison maquette - Éléments manquants

Quand proposer ?

À la fin d'une maquette/démo, avant d'envoyer au client, toujours proposer de lister les éléments manquants pour finaliser le vrai site.

Comment faire ?

Analyser ce qui a été utilisé en placeholder/fictif dans la maquette et lister :

  • Photos manquantes (équipe, produits, locaux, etc.)
  • Textes à remplacer (présentations, descriptions, etc.)
  • Informations pratiques (horaires, tarifs, coordonnées, etc.)
  • Éléments légaux (mentions légales, SIRET, etc.)
  • Contenus spécifiques au projet
Cette liste est spécifique à chaque projet selon ce qui a été maquetté.
