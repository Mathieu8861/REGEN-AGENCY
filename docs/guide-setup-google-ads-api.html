Guide Setup Google Ads API

Configuration de l'API Google Ads pour la synchronisation automatique des campagnes

Ce qu'on a besoin au final

  • Client ID OAuth xxxx.apps.googleusercontent.com
  • Client Secret OAuth GOCSPX-xxxxxxx
  • Refresh Token 1//0xxxxxxx
  • Developer Token xxXXxxXXxx
  • MCC ID (si applicable) 123-456-7890
  • Customer IDs clients 111-222-3333 par client
Etape 1

Creer un projet Google Cloud

~ 5 minutes
  1. Va sur la Google Cloud Console :
    https://console.cloud.google.com
  2. Connecte-toi avec le compte Google qui gere le MCC / les comptes ads
  3. En haut, clique sur le selecteur de projet (a cote du logo Google Cloud)
  4. Clique sur "Nouveau projet"
  5. Nom du projet : REGEN Agency - Ads Sync
    Organisation : laisse tel quel
    Clique "Creer"
  6. Attends quelques secondes, puis selectionne ce projet dans le selecteur
Note : Si tu as deja un projet Google Cloud, tu peux l'utiliser. L'important c'est d'activer l'API Google Ads dedans.
Etape 2

Activer l'API Google Ads

~ 2 minutes
  1. Dans ton projet Google Cloud, va dans le menu hamburger (en haut a gauche) → "API et services" → "Bibliotheque"
  2. Ou va directement ici :
    https://console.cloud.google.com/apis/library/googleads.googleapis.com
  3. Clique sur "Activer"
  4. Attends que ca se charge - tu verras un dashboard de l'API une fois active
Etape 3

Creer les identifiants OAuth 2.0

~ 10 minutes

C'est la partie la plus importante. On va creer les credentials qui permettent a notre serveur de se connecter a Google Ads.

3a. Configurer l'ecran de consentement OAuth

  1. Va dans "API et services" → "Ecran de consentement OAuth"
  2. Type d'utilisateur : "Externe" (meme si on l'utilise en interne)
    Clique "Creer"
  3. Remplis les champs obligatoires :
    • Nom de l'appli : REGEN Ads Sync
    • E-mail d'assistance : ton email
    • E-mail du developpeur : ton email
  4. Clique "Enregistrer et continuer" sur toutes les etapes suivantes (Portees, Utilisateurs test, Resume)

3b. Creer le Client OAuth

  1. Va dans "API et services" → "Identifiants"
  2. Clique "+ CREER DES IDENTIFIANTS" → "ID client OAuth"
  3. Type d'application : "Application Web"
    Nom : REGEN Ads Sync
  4. Dans "URI de redirection autorises", ajoute :
    https://developers.google.com/oauthplayground
    (On va l'utiliser a l'etape 5 pour generer le refresh token)
  5. Clique "Creer"
  6. Une popup apparait avec tes credentials :
    • ID client : quelque chose comme 123456789-abc.apps.googleusercontent.com
    • Code secret du client : quelque chose comme GOCSPX-xxxxxxxxxxxxxxxx
Important ! Note bien ces deux valeurs (Client ID et Client Secret). Tu peux aussi les retrouver plus tard dans la page Identifiants.
A noter :
Client ID = GOOGLE_ADS_CLIENT_ID
Client Secret = GOOGLE_ADS_CLIENT_SECRET
Etape 4

Obtenir le Developer Token

~ 5 minutes

Le developer token est un identifiant specifique a Google Ads (pas Google Cloud). Il se trouve dans ton compte MCC.

  1. Connecte-toi a Google Ads avec le compte MCC :
    https://ads.google.com
  2. Assure-toi d'etre sur le compte administrateur (MCC), pas un compte client
  3. Clique sur "Outils et parametres" (icone cle a molette en haut)
  4. Dans la section "Configuration", clique sur "Centre API"
  5. Tu verras ton Developer Token affiche sur cette page
Niveau d'acces du Developer Token
Par defaut, le token est en mode "Test" (acces limite a 15 000 requetes/jour, uniquement sur les comptes de test). Pour la production, il faut demander l'acces "Basic" via le formulaire de demande. Cependant, pour commencer et tester, le mode "Test" suffit si on utilise les vrais comptes lies au MCC.
Pas de Centre API ? Si tu ne vois pas "Centre API" dans les outils, c'est peut-etre que le compte n'est pas un MCC ou que l'acces API n'a pas ete active. Contacte le support Google Ads.
A noter :
Developer Token = GOOGLE_ADS_DEVELOPER_TOKEN
Etape 5

Generer le Refresh Token

~ 10 minutes

Le refresh token permet a notre serveur de s'authentifier sans intervention humaine. On va utiliser l'OAuth Playground de Google pour le generer facilement.

  1. Va sur l'OAuth Playground :
    https://developers.google.com/oauthplayground
  2. En haut a droite, clique sur l'icone engrenage (Settings) et configure :
    • Coche "Use your own OAuth credentials"
    • OAuth Client ID : colle ton Client ID de l'etape 3
    • OAuth Client secret : colle ton Client Secret de l'etape 3
    Ferme les settings.
  3. Dans le panneau de gauche "Step 1 - Select & authorize APIs" :
    Cherche et selectionne le scope :
    https://www.googleapis.com/auth/adwords
    Clique "Authorize APIs"
  4. Google va te demander de te connecter. Connecte-toi avec le compte qui a acces au MCC / comptes Google Ads.
  5. Accepte les autorisations demandees (meme si Google affiche un avertissement "appli non verifiee" - clique "Continuer")
  6. Tu reviens sur l'OAuth Playground.
    Dans "Step 2 - Exchange authorization code for tokens" :
    Clique "Exchange authorization code for tokens"
  7. Tu verras apparaitre :
    • Access token : (temporaire, on n'en a pas besoin)
    • Refresh token : 1//0eXXXXXXXXXXX...
Important ! Le refresh token n'est affiche qu'une seule fois. Copie-le immediatement et garde-le en securite. Si tu le perds, il faudra refaire cette etape.
A noter :
Refresh Token = GOOGLE_ADS_REFRESH_TOKEN
Etape 6

Recuperer les Customer IDs

~ 5 minutes

Chaque compte Google Ads a un identifiant unique (Customer ID). On en a besoin pour chaque client qu'on veut synchroniser.

  1. Dans Google Ads (ads.google.com), va dans le MCC
  2. Tu verras la liste de tous les comptes clients geres
  3. Pour chaque client, note le Customer ID (format : XXX-XXX-XXXX)
    Il est affiche en haut de page quand tu es dans un compte, ou dans la colonne "ID client" de la vue MCC
  4. Note aussi l'ID du MCC lui-meme (visible en haut a droite quand tu es au niveau MCC)
